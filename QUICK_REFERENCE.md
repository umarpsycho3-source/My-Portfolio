Create `.env` file locally:\n```env\nNODE_ENV=production\nADMIN_EMAIL=your-email@gmail.com\nADMIN_PASSWORD=Tr0ub4dour&Uygg7^*Hj9\nCONTACT_PHONE=+1-234-567-8900\nCONTACT_WHATSAPP=1234567890\nCONTACT_EMAIL=your-email@gmail.com\nALLOWED_ORIGIN=https://my-portfolio-zdd8.onrender.com\n```\n\n### 2. Add `.env` to `.gitignore`:\n```bash\necho \".env\" >> .gitignore\n```\n\n### 3. Set variables in Render dashboard:\n- Settings → Environment → Add all variables from `.env`\n- Save (auto-redeploys)\n\n### 4. Test login:\n- Use new credentials from your `.env` file\n- Should work immediately\n\n---\n\n## 🔐 Security Features at a Glance\n\n| Feature | What It Does | Status |\n|---------|-------------|--------|\n| Rate Limiting | 5 login attempts per 15 min | ✅ Active |\n| Input Sanitization | Escapes HTML to prevent XSS | ✅ Active |\n| Email Validation | Checks email format | ✅ Active |\n| Password Hashing | Scrypt with salt | ✅ Active |\n| Secure Cookies | HttpOnly, SameSite=Strict, Secure | ✅ Active |\n| Security Headers | CSP, X-Frame-Options, HSTS, etc | ✅ Active |\n| IP Detection | Tracks login attempts by IP | ✅ Active |\n| Session Timeout | Auto-logout after 12 hours | ✅ Active |\n| HTTPS Enforcement | Redirects HTTP to HTTPS | ✅ Active |\n| Error Logging | Tracks security events | ✅ Active |\n\n---\n\n## 📋 Password Requirements\n\nYour admin password MUST have:\n\n```\n✅ Minimum 8 characters\n✅ At least 1 UPPERCASE letter\n✅ At least 1 lowercase letter\n✅ At least 1 number (0-9)\n✅ At least 1 special character (@$!%*?&)\n```\n\n### Examples:\n- ✅ `MyP@ssw0rd!`\n- ✅ `Sec#re123Pass`\n- ✅ `Admin*2026&Pro`\n- ✅ `Tr0ub4dour&Uygg7^*Hj9`\n- ❌ `password123` (no uppercase, no special char)\n- ❌ `MyPassword` (no number, no special char)\n- ❌ `Pass@123` (only 8 chars, no uppercase)\n\n---\n\n## 🚫 Rate Limits\n\nIf you hit these limits, wait the specified time:\n\n| Endpoint | Limit | Window |\n|----------|-------|--------|\n| Login | 5 attempts | 15 minutes |\n| Hire Request | 10 requests | 1 hour |\n| Review | 5 submissions | 1 hour |\n| Other APIs | 100 requests | 15 minutes |\n\n---\n\n## 🧪 Quick Tests\n\n### Test Login Rate Limiting:\n1. Check password has no spaces before/after\n3. Try `NODE_ENV=production` in environment\n4. Restart: Settings → Manual Deploy\n\n### \"Too many login attempts\" (legitimate login)\n**Solution**:\n1. You hit rate limit (5 per 15 min)\n2. Try from different browser/device\n3. Wait 15 minutes\n\n### Security header test shows F grade\n**Solution**:\n1. Ensure `NODE_ENV=production` in Render\n2. Redeploy service\n3. Wait 5 minutes\n4. Test again\n\n### Getting 403 error\n**Solution**:\n1. Check file paths are correct\n2. Verify `/data` directory exists\n3. Check file permissions\n4. Review Render logs\n\n---\n\n## 🔄 Maintenance Schedule\n\n### Weekly\n- [ ] Check Render logs\n- [ ] Monitor for failed logins\n\n### Monthly\n- [ ] Review hire requests\n- [ ] Approve/delete reviews\n- [ ] Check database size\n\n### Quarterly\n- [ ] Rotate admin password\n- [ ] Update Node.js packages\n- [ ] Review security settings\n\n---\n\n## 📚 Documentation\n\nFor more details, read:\n- [SECURITY.md](./SECURITY.md) - Full security documentation\n- [DEPLOYMENT_CHECKLIST.md](./DEPLOYMENT_CHECKLIST.md) - Complete deployment guide\n- [RENDER_SETUP.md](./RENDER_SETUP.md) - Step-by-step Render setup\n\n---\n\n## ✅ Status\n\n- ✅ Security headers implemented\n- ✅ Rate limiting active\n- ✅ Input validation enabled\n- ✅ Password hashing secure\n- ✅ Session management hardened\n- ✅ HTTPS enforced\n- ✅ Documentation complete\n\n**Your portfolio is now secure!** 🎉\n
